PURPOSE.
The intent of Identification Card Standards is two fold:

To ensure that the bar code and magnetic stripe infrastructure developed by one State agency is compatible with the infrastructure developed by all other State agencies and similar agencies in other states.

To ensure that the content and format of data in bar codes and magnetic stripes on agency cards are similar so that cards with magnetic stripe and bar codes will easily adapt to use as universal identifier cards.

The American Association of Motor Vehicle Administrators (AAMVA) and the International Standards Organization's (ISO) have developed standards for the use of bar codes and magnetic stripes on identification cards. To enhance the inter-jurisdictional utility of cards, agencies must use these standards whenever possible.

STANDARDS REVIEW GROUP.
The ISPB shall establish a Standards Review Group. The Standards Review Group will:

Be responsible for coordinating ongoing compliance with national standards setting agencies.
Actively encourage inter state consortia, national standards groups, and federal agencies to develop standards compatible with Maine's Identification Card Standards.
Ensure that the State is issued and where possible agencies use a common ANSI User Code and ANSI User ID.
Establish and assign Agency Designator Numbers to agencies issuing cards.
Establish mechanism for etting (changing) the CRYPT KEY for encryption.

The Standards Review Group may recommend granting requests for a waver from Maine's Identification Card Standards, for final approval by the ISPB, when a waver is necessary for an agency to comply with the standards set by a multi-state agreement, a federal agency program, or an outsourcing agreement with a third party to both produce the cards and handle transaction processing.

STANDARDS FOR IDENTIFICATION CARDS.
State agencies shall comply with the following standards in the issuance of identification cards that use bar codes or magnetic stripes.

CARD SIZE:
Size of Card: ISO standard.

IDENTIFICATION NUMBER
The Standard Individual ID Number will be the cardholder's encrypted Social Security Number. Agencies can also have an Agency Specific Individual ID Number.
ONE DIMENSIONAL BAR CODE
Symbology Standard: Code 128.

Standard for Encryption: Standard UNIX Crypt Function compliant with the ANSI standards.

Standard for Physical Location of Bar Code: The bar code should occupy the area from 0.20 inch to 0.60 inch (0.5 cm to 1.5 cm) from the edge of the card. It should not be on the same edge as the magnetic stripe.

TWO DIMENSIONAL BAR CODE:
Standard for Physical Location of Bar Code: None.

Symbology Standard: PDF417.

Standard for Compression: None. No compression or compaction beyond Hi-Density, PDF417 standard.

Character Set: ASCII/ISO 8859-1 (256 character table) or for efficiency the 128 character subset in Text Compaction Table.

Standard for Encryption: Standard UNIX Crypt Function.

State Standard for Identification Number: Social Security Number and Alternative ID Numbers are included in the AAMVA Hi-Density Symbology Data Structure Template.

Data Structure: AAMVA Hi-Density Symbology Data Structure Template. The general structure is outlined below. (A more detailed description is found in the "Catalog of Defined Subfile and Data Element Identifiers Used with AAMVA Compliant PDF417 Bar Codes".)

AAMVA Hi-Density Symbology Data Structure Template:
HEADER:
The header contains:
Compliance Indicator.
Data Element Separator.
Record Separator.
Segment Terminator.
File Type: AAMVA.
Jurisdiction Code: Obtained from AAMVA.
Version Number: Version level of AAMVA Hi-Density bar code format.
Number of Entries: Specifies the number of subfiles (1-99).

SUBFILE DESIGNATOR(s):
Lists Subfile Types, Offset, and Length. The Offset indicates the start of the subfile by the number of bytes from beginning of file and Length indicates the number of bytes in the subfile.

SUBFILE: Subfiles include a Subfile Identifier followed by Data Element Identifiers.

DATA ELEMENT IDENTIFIERS: Data Element Identifiers (DEI) provide a label for the data string that follows.

DATA: The data can be either text data or contain Binary Large Objects (BLOBs).

General: AAMVA has developed a list of Subfiles Types and Data Element Identifiers. The Standards Review Group will review and approve additional Subfile Types or Data Element proposed by agencies. The Group will submit these additional designators to AAMVA for approval and registration. Agencies and the Group should foster a single set of standard designators. AAMVA has reserved Subfile Type and Data Element Identifiers for use by individual jurisdictions.

Data structure must follow ANSI D-20 standard where a definition exists. If no D-20 entry exists for a required data element, an agency, through the Review Group, should promote that element as an addition to the D-20 standard.

A Data Element Separator, a Record Separator, or Segment Terminator follows each data element depending upon whether the next field is an additional data element, record or subfile.

MAGNETIC STRIPES
Standard for Physical Location of Stripe: ISO standard. It should not be on the same edge as the one dimensional bar code.

Coercivity: None. (ISO standards are recommended.)
Encoding standard: Biphase F2F (or Biphase) Encoding Technique.

Data structure: ANSI D-20 standard.

Encryption: Standard UNIX Crypt Function.

Magnetic Stripe Field Layouts:
Track 1: ISO standard.
Track 2: AAMVA or ISO standards as indicated table below.
Track 3: ISO standard.

General: Magnetic stripes are widely used by financial institutions and the State Standard must comply as closely as possible to the general ISO standards. ISO has approved an alternative Track 2 standard for use with driver licenses. Track 2 is primarily used for identifying the cardholder and should either comply with the ISO Standard or the AMVAA Alternative (see table below). AAMVA has also proposed alternative standards for Tracks 1 and 3. However, unless magnetic stripe becomes a widely used standard for interstate motor vehicle use, the State should comply with the ISO standards. The Standards Review Group will monitor the evolving use in the motor vehicle sector.
